`git diff HEAD`를 읽고 아래 항목을 점검하세요:

1. **타입 안전성** — any 사용, 누락된 타입 정의
2. **에러 처리** — try/catch 누락, 빈 catch 블록
3. **보안** — SQL 인젝션, XSS, 하드코딩된 시크릿
4. **성능** — N+1 쿼리, 불필요한 re-render
5. **테스트** — 순수 함수에 테스트 없음

각 항목별로 문제점과 개선 방법을 한국어로 설명하세요.
심각도: 🔴 차단 / 🟡 권고 / 🟢 선택
